## 5. ID formats — corrected

### What the application actually uses

IDs are generated by `idGenerator.js` from `tblIDSequences` prefixes in `setupDefaults.js`:

| Entity | Prefix | Pattern | Example |
|--------|--------|---------|---------|
| Branch | `BR` | `BR` + 3 digits | `BR001` |
| Asset | `ASS` | `ASS` + 3+ digits | `ASS121` |
| Vendor | `V` | `V` + 3+ digits | `V047` |
| Department | `DPT` | `DPT` + 3 digits | `DPT001` |
| User | `USR` | `USR` + 3+ digits | `USR001` |
| Job role nav | `JRN` | `JRN` + 3+ digits | `JRN001` |

**Note:** The earlier audit incorrectly expected `BRN`, `AST`, and `VEN` prefixes. Those were never used by the generator — `ASS121` and `V047`/`V048` on skasc were already correct.

### SKASC fix applied

| Before | After | Action |
|--------|-------|--------|
| `BRANCH001` | `BR001` | Legacy bad branch ID remapped; `tblDepartments.branch_id` updated |
| `branch_code` = `BRANCH001` | `HO` | Standard head-office code |
| `tblIDSequences.branch` out of sync | `last_number = 1` | Synced from actual data |

Details in **Section 5** above.
